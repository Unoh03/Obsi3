# VS Code Remote-SSH 설치 중단 오류 복구 명령어

## 상황

VS Code Remote-SSH 접속 시 비밀번호는 맞지만 접속이 완료되지 않거나, 서버 설치가 계속 진행 중인 것처럼 멈추는 문제가 발생했다.

원인은 GNS 환경이 느려 VS Code Server 다운로드가 오래 걸리는 상태에서, 사용자가 VS Code를 강제 종료하면서 서버 쪽에 설치 락 또는 설치 잔재가 남는 것이다.

대표 증상:

```text
Installation already in progress
exitCode==192
```

## 1. 로컬 VS Code 완전 종료

Windows에서 VS Code 창을 모두 닫는다.

필요하면 작업 관리자에서 아래 프로세스를 종료한다.

```text
Code.exe
ssh.exe
```

## 2. 일반 SSH로 WEB 서버 접속

PowerShell 또는 Git Bash에서 실행한다.

```bash
ssh WEB
```

또는 직접 접속한다.

```bash
ssh webuser@WEB_SERVER_IP
```

## 3. 서버에서 VS Code 관련 프로세스 확인

WEB 서버에 접속한 뒤 실행한다.

```bash
ps -u "$USER" -f | grep -E 'vscode|server' | grep -v grep
```

## 4. VS Code Server 프로세스 종료

```bash
pkill -u "$USER" -f vscode-server 2>/dev/null || true
pkill -u "$USER" -f ".vscode-server" 2>/dev/null || true
```

## 5. VS Code Remote-SSH 락 파일 삭제

```bash
rm -f /run/user/$(id -u)/vscode-remote-lock.$USER.*
```

## 6. 깨진 VS Code Server 설치 폴더 삭제

```bash
rm -rf ~/.vscode-server/bin/*
```

필요하면 전체 `.vscode-server`를 지운다.

```bash
rm -rf ~/.vscode-server
```

## 7. 서버에서 나가기

```bash
exit
```

## 8. VS Code에서 다시 Remote-SSH 접속

VS Code를 다시 열고 `WEB`에 접속한다.

이때 GNS 환경이 느리면 다운로드가 오래 걸릴 수 있으므로, 설치 중에는 강제 종료하지 않는다.

## 9. 반복 발생 시 VS Code 설정 추가

로컬 VS Code `settings.json`에 추가한다.

```json
{
  "remote.SSH.useFlock": false
}
```

## 10. 빠른 복구용 전체 명령어

WEB 서버에 일반 SSH로 접속한 뒤 아래를 한 번에 실행한다.

```bash
ps -u "$USER" -f | grep -E 'vscode|server' | grep -v grep

pkill -u "$USER" -f vscode-server 2>/dev/null || true
pkill -u "$USER" -f ".vscode-server" 2>/dev/null || true

rm -f /run/user/$(id -u)/vscode-remote-lock.$USER.*
rm -rf ~/.vscode-server/bin/*

echo "VS Code Remote-SSH server cleanup complete."
```

## 판단 기준

|결과|의미|
|---|---|
|일반 `ssh WEB` 접속 성공|비밀번호/SSH 인증 문제 아님|
|VS Code만 실패|Remote-SSH 서버 설치 잔재 또는 락 문제 가능성 높음|
|`Installation already in progress`|기존 설치 프로세스 또는 락이 남은 상태|
|정리 후 재접속 성공|`.vscode-server` 설치 잔재 문제가 원인|
