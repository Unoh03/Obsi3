### PDF 등급 표기

| 표기 | 의미 |
| --- | --- |
| `상` | PDF 체크리스트 등급 상 |
| `중` | PDF 체크리스트 등급 중 |
| `하` | PDF 체크리스트 등급 하 |
| `확인 필요` | 아직 PDF 세부 항목에서 등급을 확인하지 못함 |

- 후보 수집 단계에서는 등급으로 후보를 버리지 않는다.
- PDF 등급은 수집 필터가 아니라 우선순위 판단용 속성이다.
- 실제 채택 여부는 기존 경험, 구현 난이도, 시연성, 보고서 가치, 환경 제약, 비용/장애 위험을 함께 본다.

## 우선순위 판단 기준

후보는 우선 많이 모은다. PDF 체크리스트 등급이 `상`, `중`, `하` 중 무엇이든 후보에서 버리지 않고, 등급은 후보표에 남긴다. 이 표의 목적은 후보를 줄이는 것이 아니라, 나중에 회의할 때 무엇부터 잡을지 빠르게 정하기 위한 분류다.

| 우선순위 | 기준 | 의미 |
| --- | --- | --- |
| `P0` | 기존 경험 있음 + 시연 쉬움 + 보고서 가치 높음 | 바로 가져다 프로젝트 핵심 시연으로 쓰기 좋음 |
| `P1` | 새 구현이 필요하지만 난이도 낮거나, 기존 경험과 가까움 | 시간만 있으면 핵심 후보로 올릴 수 있음 |
| `P2` | 설계/점검/방어 중심이거나 환경 구성이 더 필요함 | 보고서와 아키텍처 설명에 좋고, 시연은 선택 |
| `P3` | 난이도 높음, 위험도 높음, 직접성 낮음, 또는 PDF 매핑 약함 | 심화 예시나 시간 남을 때 볼 후보 |

| 판단 요소 | 보는 기준 |
| --- | --- |
| PDF 등급 | `상`, `중`, `하`, `확인 필요`. 수집 필터가 아니라 우선순위 판단용 속성 |
| 경험 | 이미 실습했는가, 비슷한 실습을 했는가, 완전 신규인가 |
| 난이도 | 취약 앱/인프라/라우터 설정을 얼마나 새로 만들어야 하는가 |
| 시연성 | 화면, 패킷, 로그, 요청/응답으로 눈에 보이게 증명 가능한가 |
| 보고서 가치 | PDF 항목명과 잘 맞고, 개선 전/후를 쓰기 좋은가 |
| 환경 | AWS, GNS, VM, Web, DB 중 어디가 필요한가 |
| 위험/비용 | 실제 서비스 장애, 과금, 계정 보안 위험이 큰가 |

## 추가 발굴 대기 후보

아래는 아직 원장에 충분히 풀어 넣지 않은 확장 영역이다. 기존 DBMS, UNIX/Server, App/Ops 후보는 2026-06-08 원장화 과정에서 `후보 풀`에 1차 편입했다.

| 축 | 추가 발굴 방향 | 예상 우선순위 | 이유 |
| --- | --- | --- | --- |
| 웹 서비스 | Tomcat/웹서버 설정 항목 추가 발굴 | P1/P2 | 웹 취약점과 서버 설정 취약점을 연결할 수 있음 |
| UNIX/Server | EC2/Linux hardening 항목 추가 발굴 | P1/P2 | SSH, 포트 제한, 로그, NFS 접근통제 외에도 후보가 많음 |
| DBMS | MariaDB/MySQL에 직접 맞는 DBMS 항목 선별 | P1/P2 | PDF는 Oracle/MSSQL 예시가 섞여 있어 현재 DB 엔진 기준 재선별 필요 |
| 네트워크/GNS | N-01~N-38 전체 중 구현 쉬운 항목 추가 분류 | P1/P2 | 현재는 프로젝트와 직접 맞는 항목 위주로 1차 편입함 |
| 클라우드 | CA-01~CA-19별 AWS 실제 증거 수집 방식 정리 | P1/P2 | 콘솔 캡처, IAM/SG/CloudWatch/백업 증거 방식이 필요 |
| App/Ops | Secret 분리, 배포 권한, 자동 복구를 PDF 항목에 재매핑 | P0/P1/P2 | 실전 가치가 높지만 PDF 직접 항목명은 더 확인해야 함 |

## 현재 판단

- 2026-06-08 보정: 더 이상 `상` 항목만 고르지 않는다. 체크리스트 후보는 `상/중/하`를 모두 긁어오고, 후보표와 일일 로그에는 등급을 남긴다.
- `후보 풀`이 원장이고, `우선순위 초안`은 1차 추천 후보를 보는 요약 뷰다.
- 후보를 줄이는 기준은 `기존 실습 여부`가 아니라 `프로젝트에서 시연/설계/보고서로 쓸 수 있는가`이다.
- 현재 단계에서는 후보를 많이 남긴다. 대신 경험, 난이도, 시연성, 보고서 가치, 필요 환경으로 우선순위를 분류한다.
- 기존 실습이 있는 항목은 재사용 가능성이 높지만, 보고서에서는 PDF 항목명에 맞춰 다시 이름을 붙여야 한다.
- PDF에 직접 항목명으로 없는 실습은 버리지 않고, 심화 시연이나 공격 예시로 둔다.
