## 구현·데이터·증거 위치

- `kisa-webapp-checker/checker.py` - 선언형 check를 실행해 evidence와 report를 생성하는 수집기
- `kisa-webapp-checker/checks/` - KISA Web Application 항목별 점검 정의
- `kisa-webapp-checker/profiles/` - 대상 환경, route, allowlist, source_root 설정
- `kisa-webapp-checker/payloads/` - 항목별 payload 데이터
- `kisa-webapp-checker/mock_targets/` - 로컬에서 checker 동작만 검증하는 mock target
- `kisa-webapp-checker/evidence/` - 실행 후 `evidence/<run_id>/result.json`, `report.md`, request/response를 두는 위치. 최종 판정 문서와 구분한다.

## 운영 메모

- 설계·단계·결정은 설계 노트에, 실제 실행 관찰과 검증 결과는 작업 로그에 기록한다.
- checker는 최종 판정기가 아니라 evidence collector다. README와 방향 전환 노트의 제출 범위에 따라 R4 runtime, AI review 자동화, 전체 21개 항목 검증 완료처럼 표현하지 않는다.
