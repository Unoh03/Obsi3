## 결정한 것

| 결정 | 이유 | 영향 |
|---|---|---|
| Promtail 설정 경로는 `/etc/promtail/config.yml`로 유지 | service의 `ExecStart`와 실제 설정 파일을 일치시켜야 함 | `local/config.yaml` 혼선 제거 |
| Promtail 목적지는 실제 Loki 수신 서버 기준으로 맞춤 | `connection refused`는 파일 없음이 아니라 목적지/수신 상태 문제였음 | Log/Monitor 역할과 IP를 정확히 구분해야 함 |
| Grafana의 `1.1.1.10:9100`은 Promtail 로그가 아니라 Node Exporter metrics로 분리 | Prometheus metrics 흐름과 Loki log 흐름이 다름 | 장애 원인 추적 시 Prometheus와 Loki를 섞지 않음 |
| 팀원이 완료한 항목은 팀 프로젝트 완료로 처리 | 팀 프로젝트 산출물이므로 개인 실습 여부로 분리하면 상태가 왜곡됨 | 후보표에서 `참고`가 아니라 `완료`로 표시 |
