2조 NAT 포함 IP 계획표

기준: 서버와 클라이언트에는 192.168.x.x 사설 IP를 넣고, 라우터 NAT를 통해 외부에서는 1.2.x.x 실습용 공인 역할 IP로 보이게 한다.

용어 정리: ip nat inside는 “사설 IP가 있는 안쪽 구간”, ip nat outside는 “NAT 변환 후 나가는 바깥 구간”이라는 뜻이다. 이번 구조에서는 서버/클라이언트가 붙은 LAN 쪽이 inside, Transit-SW가 있는 1.2.123.0/24 쪽이 outside다.

전체 구조

아래 그림은 토폴로지 이미지와 맞춰서 표현한 구조다. 핵심은 C-PC의 Transit-SW가 A-R, C-R, B-R을 같은 Transit LAN으로 묶는다는 점이다.

[A-PC / EDGE-DNS-DHCP] [C-PC / APP + Transit-SW] [B-PC / DATA-MGMT] Client / DHCP / DNS LB / WEB / NFS / Cache DB / Log / Monitor 사설망: 192.168.1.0/24 사설망: 192.168.2.0/24 사설망: 192.168.3.0/24 | | | Switch-A APP-SW Switch-B | | | A-R ------------------ UDP Tunnel ---------------- Transit-SW ---------------- UDP Tunnel -------- B-R | | | A-R Transit: 1.2.123.1 C-R Transit: 1.2.123.2 B-R Transit: 1.2.123.3 / C-R C-R LAN: 192.168.2.254 Transit LAN: 1.2.123.0/24 - A-R: 1.2.123.1 - C-R: 1.2.123.2 - B-R: 1.2.123.3
해석: A-R → C-R → B-R로 일렬 라우팅되는 구조가 아니다. A-R, C-R, B-R 세 라우터가 C-PC의 Transit-SW에 함께 연결되어 있고, 같은 1.2.123.0/24 Transit LAN을 공유하는 구조다.

NAT inside / outside를 쉽게 말하면

구분이번 프로젝트에서의 의미예시
ip nat inside 사설 IP를 쓰는 서버/클라이언트가 있는 쪽 A LAN 192.168.1.0/24, C APP LAN 192.168.2.0/24, B DATA/MGMT LAN 192.168.3.0/24
ip nat outside NAT 변환된 IP가 나가는 공용 Transit 쪽 Transit LAN 1.2.123.0/24

예를 들어 DNS1은 실제 서버 IP가 192.168.1.3이지만, 외부에서는 1.2.1.3으로 접근하게 만든다.

DNS1 실제 IP: 192.168.1.3 | A-R ← 여기서 NAT 변환 | 외부에서 보는 IP: 1.2.1.3
