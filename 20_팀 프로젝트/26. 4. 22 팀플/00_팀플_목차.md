## 기준 산출물과 스크립트 경계

- [[20_팀 프로젝트/26. 4. 22 팀플/토폴로지와 IP NAT ACL 기준표_(4.30).html|토폴로지와 IP NAT ACL 기준표(4.30)]] - 복구 문서가 네트워크 지도 기준으로 삼는 기준표
- `쉘 스크립트/A-R_Final.ios`, `쉘 스크립트/B-R_Final.ios`, `쉘 스크립트/C-R_Final.ios` - 현재 router final 후보. `쉘 스크립트/구닥다리/`의 날짜별 IOS는 이전 후보로 본다.
- `쉘 스크립트/LB.sh`, `쉘 스크립트/dns1.sh`, `쉘 스크립트/dns2.sh`, `쉘 스크립트/lb-upstream-update(5.6).sh` - LB/DNS 운영·갱신 후보
- `쉘 스크립트/webCompZzinFinal.sh` - 사용자 확인 기준 마지막 WEB 통합 스크립트. SMTP 587 정책 라우팅이 추가된 현재 기준으로 본다.
- `쉘 스크립트/web-comp_final.sh` - 이전 final 후보. `webCompZzinFinal.sh` 확정 이후에는 현재 실행 기준으로 단정하지 않는다.
- `쉘 스크립트/구닥다리/web-comp_5_9_.sh` - `web-comp(5.9).sh 설명서`가 설명하는 보존된 5.9 스크립트
- `쉘 스크립트/nfs-ha(5.6).sh`, `쉘 스크립트/web-nfs(5.6).sh` - `NFS 운영 매뉴얼(4.29.1)`의 NFS 서버/WEB mount 기준 스크립트
- `쉘 스크립트/구닥다리/db.sh`, `db-email-test(5.7).sh`, `web-email-test(5.7).sh`, `promtail-client*.sh`, 이전 `web-*` 계열 - 보존된 초기·테스트·이전 후보. DB/log/email 현재 운영 기준으로 단정하지 않는다.
- `구닥다리/`, `쉘 스크립트/구닥다리/` - 이전 버전과 검토 흔적. 복구 기준이나 현재 실행 대상으로 쓰기 전에 상위 문서와 실제 파일명을 다시 확인한다.
