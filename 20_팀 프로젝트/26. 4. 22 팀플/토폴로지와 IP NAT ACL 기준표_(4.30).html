데이터 흐름

흐름의미
Prometheus -> 관측 대상 node_exporter메트릭 Pull. 기본 포트 사용 시 대상 서버 9100/tcp 접근 필요
WEB1/WEB2 로그 agent -> Loki로그 Push. B-R ACL 기준 WEB 공인 역할 IP에서 Log 1.2.3.3:3100만 접근 허용
Grafana -> Prometheus대시보드에서 메트릭 조회. 같은 Monitor 서버 내부 통신이면 외부 ACL 영향 작음
Grafana -> Loki대시보드에서 로그 조회. 같은 B Zone이므로 Log 사설 IP 192.168.3.3:3100 사용 권장
B Zone 관리자 PC -> Grafana대시보드 접속. Monitor 사설 IP 192.168.3.4:3000 사용, 외부 Static NAT 불필요
관측성 트래픽은 현재 팀 결정/구현 확인이 더 필요하다. 현재 ACL상 WEB1/WEB2 -> Log Loki 3100/tcp 흐름만 허용하며, WEB 이외 서버 로그까지 Loki로 보내려면 출발지별 ACL을 추가 검토해야 한다.

주의 / 보류 항목

주의

dns2.sh의 A 레코드는 현재 dns1.sh와 같이 1.2.2.10을 바라본다. 다만 slave zone의 실제 서비스 파일은 /var/cache/bind/db.zzaphub.com이고, 스크립트가 점검하는 파일은 /etc/bind/db.zzaphub.com이라 두 파일 기준이 달라질 수 있다.
현재 C-R ACL은 LB VIP뿐 아니라 LB1/LB2 HTTP 접근도 허용한다. “사용자는 반드시 VIP만 접근”을 엄격히 만들려면 LB1/LB2 직접 접근 허용을 나중에 줄여야 한다.
