DB - Table

회원 - PRIMARY KEY(id), pw, 이름, 이메일, 전화번호, point, 칭호, 관리자 유무,이메일 인증 여부 - 초기값 False

게시글 -  PRIMARY KEY(No) , 작성자_id , 조회수 , 좋아요,싫어요, 제목, 내용, 작성일, 파일 제목, 게시글 종류

칭호 - PRIMARY KEY(no), 내용,

댓글 - PRIMARY KEY(no), 댓글_작성자_id, 게시글 id, 좋아요,싫어요, 내용,

답글 - PRIMARY KEY(no), 답글_작성자_id ,게시글 id, 좋아요, 싫어요, 내용, 댓글(답글) no

신고 - PRIMARY KEY(no),게시글 id, 신고 User_id, 신고 내용, 신고 종류

블랙리스트 - PRIMARY KEY(no), 이메일

이모티콘 - 이미지, id, userid, 관리자 허가_거부, Point

파일 저장소 - PRIMARY KEY(no), 업로더_id, 파일 제목, 게시글 종류, 파일 종류


주요 컬럼 (PK는 굵은 글씨)

Users - 회원 정보
id(회원번호), login_id, pw, name, email, phone, point, role(ADMIN/USER), is_verified

Title - 칭호 마스터
id(칭호번호), name(칭호명)

User_Title - 회원 보유 칭호
user_id, title_id, is_equipped(착용여부)

Blacklist - 가입 차단
id, email(차단 이메일), reason, created_at



커뮤니티 콘텐츠 도메인

Post - 게시글
id, user_id(FK), category(분류), title, content, view_count, status(정상/블라인드), created_at

Comment-  댓글 & 답글

id, post_id(FK), user_id(FK), parent_id(FK-답글용), content, status, created_at

File - 첨부 파일
id, target_id(게시글/댓글ID), uploader_id, origin_name, nfs_path(저장경로), file_type

상호작용 및 운영 도메인

Reaction - 좋아요/싫어요
id, user_id, target_type(POST/COMMENT), target_id, reaction_type(LIKE/DISLIKE)

Report - 신고 내역
id, reporter_id, target_type, target_id, reason, status(처리대기/완료)4. 상점 도메인테이블명역할주요 컬럼Emoticon이모티콘 상품id, creator_id, name, nfs_image_path, price, status(승인대기/판매중/반려)User_Emoticon회원 구매 내역user_id, emoticon_id, purchased_at
