# 자료 목차

## 자료 역할 경계

- 강의 원본 PDF는 source material이다. 학습 재개는 먼저 `10_학습 노트/`의 관련 MOC에서 시작하고, 원문 확인이 필요할 때만 이 목차로 내려온다.
- Source Digest는 PDF/OCR 기반 변환본이다. 원문 보존과 AI 탐색을 돕지만, stable concept note로 단정하지 않는다.
- 프로젝트 안내서·취약점 평가 가이드·이전 기수 자료는 프로젝트나 보고서의 근거 자료다. 현재 작업 기준은 관련 프로젝트 MOC에서 다시 확인한다.
- 캡처 창고는 이미지 보관소다. 전체 폴더를 훑지 말고, 노트의 이미지 링크나 파일명 단서가 있을 때만 연다.
- `.pkt`, `.sh` 같은 실습 파일은 원본 자산이다. 현재 운영 스크립트나 최종 산출물 여부는 프로젝트 MOC와 보고서에서 판정한다.

## 보관 규칙

- 강의 원본 PDF와 실습 원본 파일은 `40_자료/강의 자료/`에 보관한다.
- 노트에서 반복 참조하는 캡처만 유지하고, 임시 캡처는 정리 대상으로 둔다.
- 대용량 산출물이나 재생성 가능한 파일은 Git에 넣기 전에 필요성을 먼저 확인한다.
- source digest와 stable note가 충돌하면 stable note를 우선하지 않는다. 먼저 source digest의 성격과 원본 PDF 재대조 필요 여부를 확인한다.
