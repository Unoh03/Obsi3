# Obsi3

Obsi3는 클라우드 보안 엔지니어링 공부를 위해 만든 Obsidian vault입니다.

리눅스, 네트워크, 시스템 보안, 웹 보안, 자격증, 팀 프로젝트, 실습 기록, 강의 자료, 캡처 이미지, 템플릿을 한곳에 모아 둔 개인 학습 저장소입니다.

이 저장소는 일반 문서 사이트라기보다 **Obsidian으로 열었을 때 가장 잘 보이는 노트 묶음**입니다.

## 읽는 방법

### 가장 추천하는 방법: Obsidian으로 열기

1. 이 저장소를 clone 또는 download합니다.
2. [Obsidian](https://obsidian.md/)을 설치합니다.
3. 저장소 폴더를 Obsidian vault로 엽니다.
4. [Home](00_index/Home.md)부터 보면 됩니다.

Obsidian으로 열면 wiki link, 이미지 embed, 첨부 파일, backlinks, graph view, 폴더별 목차를 훨씬 편하게 볼 수 있습니다.

### GitHub에서 그냥 보는 경우

GitHub에서도 대부분의 Markdown 파일은 열 수 있지만, Obsidian 전용 기능은 완전히 동작하지 않을 수 있습니다.

- Obsidian wiki link가 GitHub 일반 링크처럼 보이지 않을 수 있습니다.
- 이미지, PDF, 로컬 첨부 파일을 따라가기가 불편할 수 있습니다.
- backlinks, graph view, templates, plugin 기능은 Obsidian 기준입니다.
- 일부 노트는 완성된 글이 아니라 실습 로그, 수업 메모, 초안입니다.

GitHub에서 링크가 이상하게 보이면 Obsidian으로 vault를 열고 [Home](00_index/Home.md)부터 따라가는 편이 낫습니다.

## 노트 종류

이 vault에는 여러 성격의 노트가 섞여 있습니다.

- 개념 노트: XSS, 쿠키, HTTP, Linux 권한, 네트워크 보안 같은 재사용 가능한 개념 정리
- 실습 노트: 명령어, 스크린샷, 실수, 수정 과정, 최종 관찰을 포함한 실습 기록
- 트러블슈팅 노트: 증상, 가설, 확인한 명령, 원인, 수정, 검증 기록
- 목차/MOC 노트: 폴더나 주제별로 노트를 찾아가기 위한 지도
- 자료 노트: PDF, 이미지, 스크립트, 실습 파일 등 원본 자료를 연결하는 노트

## 공유 시 주의사항

이 vault는 원래 개인 공부용으로 만든 저장소라서, 일부 파일은 수업 맥락을 알아야 이해하기 쉽습니다.

자료를 다시 사용하거나 공유하기 전에 해당 파일이 개인 정리인지, 강의 자료인지, 스크린샷인지, 외부 자료인지 확인해야 합니다. 특히 강의 PDF와 실습용 다운로드 파일은 별도의 저작권이나 수업용 제한이 있을 수 있습니다.

오래된 실습 노트나 캡처 이미지에 비밀번호, 토큰, 쿠키, 세션 ID 같은 값이 보이더라도 재사용 가능한 예시로 취급하면 안 됩니다.
