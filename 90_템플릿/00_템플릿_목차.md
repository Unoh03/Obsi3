# 템플릿 목차

## 사용 절차

1. 새 노트의 역할을 먼저 정한다.
2. 아래 표에서 가장 가까운 템플릿 하나만 적용한다.
3. `type`, `status`, `created`와 라우팅 필드를 채운다.
4. 쓰지 않는 선택 필드와 본문 섹션은 삭제한다.
5. 새 wiki link는 target을 확인한 뒤 작성한다.
6. 저장소 작업은 `python scripts/validate_frontmatter.py --changed`로 확인한다.

Obsidian에서는 `Templates: Insert template` 명령을 사용한다. 반복해서 쓴다면 이 명령에 단축키를 지정한다.

## Frontmatter 표준

모든 새 노트는 `type`, `status`, `created`를 가진다. 나머지는 역할에 맞을 때만 쓴다.

| 필드 | 용도 |
|---|---|
| `type` | 문서 역할. 진행 상태를 넣지 않는다. |
| `status` | `draft`, `active`, `stable`, `legacy`, `stale`, `archived` 중 하나 |
| `created` | 노트 생성일 |
| `topic` | 학습 주제 |
| `parent_moc` | 학습·자료 노트의 상위 MOC |
| `project`, `project_moc` | 프로젝트와 상위 프로젝트 MOC |
| `source`, `source_pages` | 원자료와 범위 |
| `raw_log`, `source_raw` | 정리의 근거가 된 RAW 로그 |
| `evidence` | 실행 결과·캡처·proof 위치 |
| `environment`, `verified_on` | 재현 환경과 실제 검증일 |
| `reviewed_on` | 내용 검토일. `reviewed`를 status로 쓰지 않는다. |

## 상태 규칙

- `raw`는 `type` 또는 RAW 관계 필드로만 쓰고 status로 쓰지 않는다.
- `complete`는 새 status로 쓰지 않는다. 재사용 가능한 정리본은 `stable`, 종료 기록은 `archived`다.
- `reviewed`는 `reviewed_on`에 날짜로 기록한다.
- `status`를 `태그/상태`로 중복 저장하지 않는다.
- 기존 노트는 전수 소급하지 않는다. 직접 수정하거나 현재 경로에 올릴 때만 점진적으로 정렬한다.
