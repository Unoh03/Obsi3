## 운영 기준

- 템플릿의 빈 bullet, blockquote, 표 셀은 작성 자리표시자다. 전체 trailing whitespace 제거 대상으로 보지 않는다.
- 변경분은 `git diff --check`로 검증하고, target 없는 빈 wiki link는 만들지 않는다.
- 프로젝트 RAW/일일로그는 source 자체이거나 digest 대상이므로 학습 노트의 `source`/`raw_log` 필드를 그대로 강제하지 않는다.
- 프로젝트 템플릿의 기본 위상은 `RAW 로그 = source`, `일일 로그 = digest`, `프로젝트 문서 = stable document`, `회의록 = decision/action source`로 구분한다.
- 새 인덱스/MOC는 `type: moc`, `scope`, `parent_moc`, `status`를 기본 표본으로 쓴다. 기존 MOC에는 frontmatter를 전수 소급하지 않는다.

## 템플릿 선택 예시

| 상황 | 사용할 템플릿 | 주의 |
|---|---|---|
| 나중에 다시 읽을 독립 개념 정리 | `개념정리_템플릿.md` | 명령 출력이나 실습 증거는 별도 lab note로 분리한다. |
| 실제 명령, 화면, 검증 결과가 있는 실습 | `실습기록_템플릿.md` | 진행 중 기록이면 RAW 여부를 노트 경계에 남긴다. |
| 반복해서 재사용할 명령어 묶음 | `명령어정리_템플릿.md` | `verified_on`과 `environment`를 비워 둔 채 현재 기준으로 단정하지 않는다. |
| 오류 원인과 해결 절차 복구 | `트러블슈팅_템플릿.md` | 증상, 원인, 해결, 재발 방지를 분리한다. |
| 새 목차나 주제 지도 | `인덱스_MOC_템플릿.md` | 전체 파일 목록이 아니라 재시작 경로와 source/RAW 경계를 좁힌다. |
| 프로젝트 중 발생한 원문 작업 로그 | `프로젝트_RAW_로그_템플릿.md` | 요약하지 말고 source 로그로 남긴다. |
| 하루 작업을 보고서 재료로 요약 | `프로젝트_일일_로그_템플릿.md` | 원문 RAW를 `source_raw`로 연결한다. |
| 최종 산출물, 설계, 보고서형 문서 | `프로젝트문서_템플릿.md` | 근거 RAW와 evidence를 분리한다. |
| 회의 결정과 후속 액션 | `회의록_템플릿.md` | 결정, 액션, 근거를 섞지 않는다. |
| 시험 오답 복구 | `오답노트_템플릿.md` | 왜 틀렸는지와 다음 회상 단서를 분리한다. |

## Properties 필드 표준

모든 템플릿에 모든 필드를 강제하지 않는다. 아래 표는 새 노트 작성 시 같은 의미의 필드를 섞어 쓰지 않기 위한 기준이다.

| 필드 | 사용 대상 | 의미 |
|---|---|---|
| `type` | 전체 | 노트 역할. 예: `concept`, `lab`, `troubleshooting`, `moc`, `project-raw` |
| `topic` | 학습 노트 | 학습 주제 축. 프로젝트 문서에는 강제하지 않는다. |
| `project` | 프로젝트 문서 | 프로젝트 축. 학습 노트에는 강제하지 않는다. |
| `status` | 전체 | `draft`, `active`, `archived`, `legacy` 같은 현재 위상 |
| `parent_moc` | 학습 노트 | 상위 학습/MOC 경로 |
| `project_moc` | 프로젝트 문서 | 상위 프로젝트 MOC 경로 |
| `source` | 학습·회의·정책 노트 | 원자료, 강의자료, 회의 원문, 참고 자료 |
| `raw_log` | 실습 노트 | 실습 중 남긴 RAW 로그나 작업 메모 |
| `source_raw` | 프로젝트 digest/stable 문서 | 요약·정리의 근거가 되는 프로젝트 RAW 로그 |
| `evidence` | 검증·프로젝트 문서 | 스크린샷, 명령 출력, proof 파일, 확인 근거 |
| `environment` | 실습·명령·트러블슈팅 | OS, 도구, 네트워크, 계정 권한 등 재현 조건 |
| `verified_on` | 명령·검증 노트 | 명령이나 절차를 실제 확인한 날짜 |
| `reviewed` | concept note | 개념 노트를 재검토한 날짜 |
| `source_pages` | PDF 기반 노트 | PDF나 교재의 쪽 범위 |
