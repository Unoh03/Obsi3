# 🔄 자바스크립트 레거시 ➡️ 모던/보안 마이그레이션 가이드

강사님의 오래된 JSP/JS 예제를 만날 때마다 아래 표를 기준으로 코드를 리팩토링(Refactoring)할 것.

| 분류 | ❌ 레거시 (강사님 코드/과거 방식) | ✅ 모던 & 시큐어 (현재 실무/보안 표준) | 멘토의 일침 (이유) |
| :--- | :--- | :--- | :--- |
| **변수 선언** | `var num = 10;` | `let num = 10;`<br>`const max = 100;` | `var`는 블록 스코프를 무시하고 호이스팅(Hoisting) 버그를 유발함. 변하는 값은 `let`, 안 변하는 값(DOM 객체 등)은 무조건 `const`를 쓸 것. |
| **문자열 조합** | `'이름: ' + name + ' 나이: ' + age` | `` `이름: ${name} 나이: ${age}` `` | 백틱(`` ` ``)을 이용한 템플릿 리터럴. 가독성이 압도적으로 좋고 줄바꿈도 그대로 인식됨. |
| **화면 출력 (1)** | `document.write('hello');` | `element.textContent = 'hello';` | `document.write()`는 기존 화면을 다 날려버림. 특정 DOM 요소를 선택해 값을 넣는 방식으로 바꿔야 함. |
| **화면 출력 (2)**<br>*(보안 핵심)* | `element.innerHTML = userInput;` | `element.textContent = userInput;` | **[XSS 방어]** `innerHTML`은 입력값을 HTML/Script로 실행해버림. `textContent`는 순수 텍스트로만 렌더링하여 악성 스크립트 실행을 원천 차단함. |
| **동등 비교** | `if (a == '10')` | `if (a === '10')` | `==` (느슨한 비교)는 `0 == false`를 `true`로 만드는 기괴한 자동 형변환을 일으킴. 타입까지 엄격하게 검사하는 `===` (일치 연산자)를 무조건 쓸 것. |
| **기본값 설정** | `let name = input || 'Guest';` | `let name = input ?? 'Guest';` | `||`는 숫자 `0`이나 빈 문자열 `""`도 거짓으로 판단해 날려먹음. Nullish Coalescing(`??`)은 `null`과 `undefined`일 때만 작동함. |
| **함수 선언** | `function add(a, b) { return a+b; }` | `const add = (a, b) => a + b;` | 화살표 함수(Arrow Function). 콜백 함수나 이벤트 리스너를 작성할 때 코드가 간결해지고 `this` 바인딩 문제가 줄어듦. |
| **배열/문자열 검색** | `if (arr.indexOf('apple') !== -1)` | `if (arr.includes('apple'))` | `indexOf`로 `-1`인지 비교하는 건 가독성이 떨어짐. 직관적인 `includes()`를 사용할 것. |
| **비동기 통신** | `XMLHttpRequest` (XHR) | `fetch('/api/data')` 또는 `axios` | XHR은 코드가 길고 콜백 지옥을 유발함. 모던 웹에서는 Promise 기반의 `fetch` API나 `async/await`를 사용함. |

>[!danger] 🚨 멘토의 특별 경고: `innerHTML`의 올바른 사용법
> `innerHTML`이 무조건 악(Evil)은 아니다. 
> - **안전한 경우:** 개발자가 직접 작성한 하드코딩된 HTML 구조를 넣을 때 (다른 채팅방 예제처럼)
> - **위험한 경우:** `name1.value` 처럼 **사용자의 입력값**이 단 1%라도 섞여 들어갈 때. 이때는 무조건 `textContent`를 쓰거나, DOM 요소(`document.createElement`)를 직접 만들어 조립해야 한다.
## 💀[RCA] 1. 왜 띄어쓰기로 문자를 붙일 수 없는가? (The Delimiter Rule)

> [!danger] 🚨 아키텍트의 팩트 폭격: "컴퓨터에게 띄어쓰기(Space)는 '단어의 끝'을 알리는 구분자(Delimiter)다. 띄어쓰기로 문자를 붙이겠다는 건, 벽돌과 벽돌 사이에 시멘트(+)를 안 바르고 그냥 허공에 띄워놓겠다는 미친 짓이다."

- **V8 엔진의 시선:** 강도님이 `"안녕" "하세요"` 라고 치면, 브라우저 엔진은 이걸 하나의 문장으로 보지 않는다. "어? '안녕'이라는 데이터가 끝났네? 근데 왜 갑자기 뒤에 '하세요'라는 알 수 없는 데이터가 또 튀어나와?" 하고 뇌 정지가 오며 **`Uncaught SyntaxError`**를 뱉고 장렬하게 뻗어버린다.
- **`+` 연산자의 진짜 의미:** `+`는 숫자일 때는 덧셈(ALU 연산)을 하지만, 문자일 때는 **"앞의 메모리 주소와 뒤의 메모리 주소를 하나로 이어 붙여라(Concatenation)"**라는 명확한 메모리 할당 명령어다.

## 🚀 [Architecture] 2. `+`가 귀찮은 자들을 위한 구원: 템플릿 리터럴 (ES6)

강도님처럼 `+`와 따옴표(`""`)를 닫고 여는 것에 빡친 전 세계 개발자들이 2015년에 새로운 문법을 창조했다. **이걸 쓰면 강도님이 원하는 '띄어쓰기'가 10000% 가능해진다.**

- **레거시 (강도님이 빡친 방식):** 
  `let text = "내 이름은 " + name + " 이고, 나이는 " + age + " 살이다.";`
- **모던 JS (백틱 ` ` 의 마법):** 키보드 물결표(`~`) 자리에 있는 **백틱(Backtick)**으로 감싸면 끝난다.
  `let text = \`내 이름은 ${name} 이고, 나이는 ${age} 살이다.\`;`
  👉 **팩트 폭격:** 백틱 안에서는 띄어쓰기, 엔터(줄바꿈)가 전부 그대로 인식된다. 변수는 `${ }` 안에 꽂아 넣기만 하면(Injection) 끝이다. (이게 현대 JS의 절대 표준이다.)

## 🛡️ [Security] 3. DevSecOps가 반드시 알아야 할 JS 3대 생존 법칙

자바스크립트는 10일 만에 대충 만들어진 언어라 태생적으로 버그가 미친 듯이 많다. 아래 3가지를 어기면 서버가 털리거나 화면이 박살 난다.

### 🚨 법칙 1: `==` 은 쓰레기다. 무조건 `===` 을 써라. (Type Coercion)
- **`==` (느슨한 비교):** `1 == "1"` 을 `true`라고 우기는 미친 연산자다. (숫자와 문자를 지 맘대로 변환해서 비교함)
- **`===` (엄격한 비교):** `1 === "1"` 은 `false`다. 값과 '데이터 타입(Type)'까지 완벽하게 일치해야만 인정한다.
- **보안 엔지니어의 시선:** 해커가 권한 체크 로직을 우회(Auth Bypass)할 때 `==`의 허술함을 찌른다. 실무에서 `==`를 쓰면 사수한테 뺨을 맞는다.

### 🚨 법칙 2: `var` 는 죽었다. `let` 과 `const` 만 써라. (Scope Hoisting)
- **`var` (레거시):** 변수를 선언하면 전역(Global)으로 오염되어, 엉뚱한 곳에서 값이 덮어씌워지는 대참사(Memory Leak)가 발생한다.
- **`let` (변수) / `const` (상수):** 자신이 선언된 중괄호 `{ }` 블록 안에서만 생존하는 안전한 샌드박스(Sandbox) 변수다.

### 🚨 법칙 3: 세미콜론(`;`)을 생략하지 마라. (ASI의 배신)
- JS는 줄 끝에 세미콜론을 안 써도 브라우저가 알아서 붙여주는 기능(ASI)이 있다.
- **팩트 폭격:** 이 멍청한 자동 완성 기능을 믿었다가, 엉뚱한 줄에서 코드가 끊겨서 프로덕션(실서버) 화면이 백지화되는 사고가 매년 일어난다. 문장이 끝나면 무조건 **`;`** 을 찍는 습관을 들여라.

 	true && true  -> true

### 📝 자바스크립트 기초 문법 예제 분석 리포트

| 파일명 | 핵심 교육 목적 (학습 목표) | 상세 내용 |
| :--- | :--- | :--- |
| **ex01** | **데이터 타입(자료형)과 `typeof`** | JS가 처리하는 원시 타입(숫자, 문자, 논리형)과 참조 타입(객체, 배열), 그리고 `undefined`와 `null`의 차이를 `typeof` 연산자로 확인. |
| **ex02** | **산술 및 증감 연산자** | `++`, `--`를 통한 변수 값의 증감 원리와 `%`(나머지), `*`(곱하기) 등 기본적인 산술 연산의 흐름 이해. |
| **ex03** | **비교 연산자와 엄격한 비교(`===`)** | 값의 대소 비교 및 **동등(`==`)과 일치(`===`)의 차이** 학습. JS 특유의 타입 변환 없이 데이터 타입까지 완벽히 비교하는 `===`의 중요성 강조. |
| **ex04** | **논리 연산자와 삼항 연산자** | `&&`(AND), `\|\|`(OR), `!`(NOT)의 진리표(Truth Table) 이해. `조건 ? 참 : 거짓` 형태의 삼항 연산자를 이용한 짧은 조건 분기 학습. |
| **ex05** | **복합 대입 연산자** | `+=`, `-=`, `*=`, `/=`, `%=` 를 사용하여 코드를 간결하게 작성하는 방법. 숫자 연산뿐만 아니라 문자열(`st += "Javascript"`) 결합에도 사용됨을 확인. |
| **ex06** | **`if` 조건문과 Falsy 값** | `if ~ else if ~ else`를 통한 다중 조건 분기. 특히 JS에서 숫자 `0`이 `false`로 취급되는 특성(Falsy)을 활용한 조건 제어 이해. |
| **ex07** | **`switch` 분기문과 내장 객체** | `Date().getDay()` 내장 함수로 요일 값을 받아, `if`문 대신 `switch ~ case`문으로 특정 값에 딱 떨어지는 분기 처리를 하는 방법. |
| **ex08** | **중첩 `for` 반복문** | `for`문 안에 `for`문을 넣는 다중 반복문 구조 이해. 구구단 출력을 통해 바깥 루프(x)와 안쪽 루프(y)의 실행 순서 파악. |
| **ex09** | **`while` 반복문** | 횟수가 정해진 `for`문과 달리, 특정 조건(`x <= 100`)이 참인 동안 계속 반복하며 값을 누적(`sum += x`)하는 로직 이해. |
| **ex10** | **함수(Function)의 정의와 호출** | 코드를 재사용하기 위한 함수의 3단계 구조 학습. 1) 단순 호출, 2) 매개변수(Parameter) 전달, 3) `return`을 통한 결과값 반환 아키텍처. |
