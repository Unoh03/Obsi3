# `사용자 데이터 - 선택 사항` 에 넣을 거
EC2 생성 화면의 `고급 세부 정보 > 사용자 데이터 - 선택 사항`에 넣는다.
새 서버가 처음 부팅될 때 root 권한으로 한 번 실행되므로 `sudo`는 쓰지 않는다.

```bash
#!/bin/bash
set -euo pipefail

SWAP_FILE="/swapfile"
SWAP_SIZE="2G"

if ! swapon --show=NAME | grep -qx "${SWAP_FILE}"; then
  if [[ ! -f "${SWAP_FILE}" ]]; then
    fallocate -l "${SWAP_SIZE}" "${SWAP_FILE}" || dd if=/dev/zero of="${SWAP_FILE}" bs=1M count=2048
  fi

  chmod 600 "${SWAP_FILE}"
  mkswap "${SWAP_FILE}"
  swapon "${SWAP_FILE}"
fi

grep -qxF "${SWAP_FILE} none swap sw 0 0" /etc/fstab || echo "${SWAP_FILE} none swap sw 0 0" >> /etc/fstab

cat >/etc/sysctl.d/99-low-memory-lab.conf <<'EOF'
vm.swappiness = 10
EOF

sysctl --system >/dev/null
```

이 스크립트의 목적은 `t3.micro`처럼 메모리가 작은 EC2에서 VS Code Remote-SSH, Tomcat, package install이 한꺼번에 돌 때 서버가 멈추는 상황을 줄이는 것이다. 이미 swap이 켜져 있으면 다시 만들지 않고 넘어간다.

> [!warning] User data는 만능 복구가 아니다
> 이 스크립트는 메모리 부족 완화용이다. Security Group, Route Table, 잘못된 Tomcat 설정, DB 연결 오류, 무한 루프 프로세스까지 해결하지는 않는다.

# EC2 서버가 멈추거나 CPU 100% 찍을 때

## 1. 로컬에서 SSH 포트 확인

```powershell
Test-NetConnection <PUBLIC_IP> -Port 22
```

```powershell
ssh -vvv -i "C:\Users\Unoh\.ssh\<KEY>.pem" ec2-user@<PUBLIC_IP>
```

## 2. SSH 세션 멈췄을 때

```text
Enter
~.
```

## 3. 재접속

```powershell
ssh -o ServerAliveInterval=15 -o ServerAliveCountMax=3 -i "C:\Users\Unoh\.ssh\<KEY>.pem" ec2-user@<PUBLIC_IP>
```

## 4. 서버 상태 확인

```bash
uptime
free -h
swapon --show
ps -eo pid,user,pcpu,pmem,comm,args --sort=-pcpu | head -15
```

## 5. Swap 2GB 추가

```bash
if ! swapon --show=NAME | grep -qx '/swapfile'; then
  sudo fallocate -l 2G /swapfile
  sudo chmod 600 /swapfile
  sudo mkswap /swapfile
  sudo swapon /swapfile
fi
```

```bash
grep -qxF '/swapfile none swap sw 0 0' /etc/fstab || echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 6. fstab 중복 제거

```bash
sudo cp /etc/fstab /etc/fstab.bak
sudo awk '!seen[$0]++' /etc/fstab | sudo tee /etc/fstab.tmp >/dev/null
sudo mv /etc/fstab.tmp /etc/fstab
grep -n '/swapfile' /etc/fstab
sudo findmnt --verify
```

## 7. VS Code Remote-SSH 서버 초기화 멈춤 정리

```bash
sudo pkill -f vscode-server || true
sudo pkill -f '.vscode-server' || true
rm -rf ~/.vscode-server ~/.vscode-server-insiders
```

## 8. VS Code 원격 초기화 고착 방지 설정

```json
{
  "window.restoreWindows": "none",
  "remote.SSH.useLocalServer": false,
  "remote.SSH.useExecServer": false,
  "remote.SSH.connectTimeout": 30,
  "remote.SSH.lockfilesInTmp": true,
  "remote.SSH.remoteServerListenOnSocket": false
}
```
