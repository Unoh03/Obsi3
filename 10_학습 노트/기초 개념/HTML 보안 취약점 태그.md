### 💀 1. 데이터 인젝션의 관문: `<form>` 과 `<input>`
- **역할:** 사용자가 아이디, 비밀번호, 검색어를 입력해서 **톰캣(Tomcat) 서버로 데이터를 쏘아 보내는(POST/GET) 유일한 합법적 통로**다.
- **보안 엔지니어의 시선 (SQL Injection):** 해커는 `<input>` 창에 `admin' OR '1'='1` 같은 미친 SQL 쿼리를 적어서 서버로 던진다. 이 태그에서 날아온 데이터는 **'절대 신뢰할 수 없는 오염된 데이터(Untrusted Data)'**로 간주하고, 백엔드에서 무조건 필터링(Sanitizing)해야 한다.

### 💀 2. 원격 코드 실행의 심장: `<script>`
- **역할:** 브라우저 안에서 자바스크립트(JS) 엔진을 구동시키는 태그다.
- **보안 엔지니어의 시선 (XSS 공격):** 해커가 게시판 글쓰기 란에 `<script> 악성코드 </script>`를 몰래 적어서 올린다. 다른 사용자가 그 글을 읽는 순간, 브라우저는 저 태그를 보고 악성코드를 실행하여 세션 쿠키(로그인 토큰)를 해커의 서버로 빼돌린다. (Cross-Site Scripting의 핵심 매개체)

### 💀 3. 차원의 문 (격리 붕괴): `<iframe>`
- **역할:** 내 웹페이지 안에 '다른 웹사이트'를 네모난 창으로 띄워주는 태그다. (유튜브 영상 퍼가기 등)
- **보안 엔지니어의 시선 (Clickjacking):** 해커가 투명한 `<iframe>`으로 은행 송금 버튼을 내 사이트 위에 겹쳐 놓는다. 사용자는 귀여운 고양이 사진을 클릭했다고 생각하지만, 실제로는 은행 송금 버튼이 눌린다. (보안 헤더인 `X-Frame-Options`로 방어해야 함)

### 💀 4. 피싱의 시작점: `<a>` (Anchor)
- **역할:** 클릭하면 다른 페이지로 이동하는 하이퍼링크 태그다. (`href` 속성 사용)
- **보안 엔지니어의 시선 (Phishing & SSRF):** 해커가 `href` 속성에 교묘하게 철자가 틀린 가짜 은행 사이트(`www.g00gle.com`)를 걸어두거나, 서버 내부망(`127.0.0.1`)을 찌르는 링크를 숨겨둔다.

### 🛡️ 5. 방어막의 사령부: `<meta>` (Head 태그 내부)
- **역할:** 화면에 보이지 않지만, 브라우저에게 문서의 인코딩이나 보안 정책을 지시하는 태그다.
- **보안 엔지니어의 시선 (CSP):** 여기에 **Content Security Policy (CSP)**를 설정하여, "우리 사이트는 `google.com`에서 가져온 스크립트만 실행하고, 나머지는 다 차단해!"라고 브라우저 커널 레벨의 방화벽을 칠 수 있다.
