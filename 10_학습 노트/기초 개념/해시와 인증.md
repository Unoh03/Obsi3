# 해시와 인증

## 한 줄 요약

해시는 데이터가 바뀌었는지 확인하는 무결성 검증에 쓰이고, 인증은 문서나 통신 주체가 정당한 존재인지 확인하는 절차다.

---

## 해시

해시는 입력 데이터를 일정 길이의 요약값으로 바꾸는 함수다.

강의자료에서는 해시를 다음 관점과 연결해 설명한다.

- 무결성 확인
- 데이터 변조 여부 확인
- MAC/HMAC 또는 전자서명 같은 인증 구조의 재료

> [!warning] 주의
> 해시값만으로는 메시지의 작성자나 송신자를 인증할 수 없다.
> 메시지 인증에는 보통 `MAC/HMAC`처럼 비밀키를 함께 쓰는 구조나, 전자서명처럼 개인키를 사용하는 구조가 필요하다.

```text
원본 데이터
→ 해시 함수
→ 해시값
```

원본 데이터가 조금만 바뀌어도 해시값은 달라질 수 있으므로, 송신 전후의 해시값을 비교해 데이터 변경 여부를 확인할 수 있다.

### 강의자료의 예시

- `MD5`
- `SHA`
- `SHA-1`
- `SHA-2`
- `SHA-3`

> [!note] 연결
> 비밀번호 저장에서 해시를 어떻게 쓰는지는 [[암호화와 해시]]에서 별도로 다룬다. 이 노트는 해시를 **무결성 검증 도구**로 보는 강의자료 흐름에 맞춘다.

---

## 무결성

무결성은 전송되거나 저장된 정보가 중간에 바뀌지 않았는지 확인하는 성질이다.

```text
송신자 해시값 = 수신자 해시값
→ 전송 중 내용이 바뀌지 않았다고 판단
```

해시는 원본 내용을 숨기는 암호화와 목적이 다르다.

| 구분 | 핵심 질문 |
| --- | --- |
| 암호화 | 제3자가 내용을 읽을 수 있는가? |
| 해시 | 내용이 바뀌었는가? |

---

## 인증

인증은 어떤 문서나 행위가 정당한 절차로 이루어졌는지, 또는 통신 주체가 누구인지 확인하는 과정이다.

강의자료는 인증기관이 인증서를 발급하고 관리하는 흐름을 함께 다룬다.

### 인증서 구성

- 발행기관
- 가입자 정보
- 유효기간
- 지문
- 공개키
- 일련번호

인증서는 단순한 파일이 아니라, **신원 정보와 공개키를 연결해주는 근거**로 볼 수 있다.

---

## 인증기관과 인증 서비스

인증기관은 인증서를 발급하고 관리하는 공신력 있는 주체다.

인증 서비스는 다음과 같은 작업을 포함한다.

- 인증서 발급
- 인증서 관리
- 인증 관련 절차 제공

강의자료 수준에서는 인증기관이 "이 공개키가 정말 누구의 것인지"를 확인해주는 중간 신뢰 주체라고 이해하면 충분하다.

---

## Kerberos

`Kerberos`는 네트워크 인증 시스템을 목적으로 개발된 대표적인 인증 프로토콜이다.

강의자료에서는 다음 정도만 다룬다.
![[40_자료/캡쳐 창고/암호화.webp]]

[[암호화.pdf#page=16&rect=43,49,736,450|암호화, p.16]]
- MIT에서 개발
- 중앙 인증 시스템
- 이름은 그리스 신화의 켈베로스에서 유래

여기서 중요한 포인트는, 인증이 단순히 비밀번호 한 번 확인하는 행위가 아니라 **네트워크 안에서 사용자를 식별하고 신뢰를 이어가는 체계**가 될 수 있다는 점이다.

## 암호화와의 관계

| 보안 목표 | 주된 연결 |
| --- | --- |
| 기밀성 | 암호화 |
| 무결성 | 해시 |
| 인증 | 인증서, 인증기관, Kerberos |

세 목표는 서로 다른 질문에 답하지만, 실제 보안 시스템에서는 함께 쓰인다.

3. 인증은 어떤 문제를 해결하는가?
4. 인증서에는 어떤 정보가 들어가는가?
5. 인증기관은 왜 필요한가?
6. Kerberos는 어떤 범주의 기술인가?
