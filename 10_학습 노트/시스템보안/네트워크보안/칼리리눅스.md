# 기초 셋팅

## BackTrack이랑 Kali Linux 차이

한 줄로 정리하면:

```text
BackTrack = Kali Linux의 전신, 구세대 보안 실습 배포판
Kali Linux = BackTrack을 계승해 새로 정리된 현재 세대 보안 실습 배포판
```

둘 다 목적은 비슷하다.

- 침투 테스트
- 취약점 분석
- 포렌식
- 보안 감사
- 네트워크 보안 실습

차이는 **지금 쓰는 표준이 무엇인가**다.

| 구분 | BackTrack | Kali Linux |
| --- | --- | --- |
| 위치 | Kali 이전 세대 | BackTrack 후속 |
| 상태 | 구버전 자료에서 자주 보임 | 현재 주로 사용하는 배포판 |
| 기반 | 오래된 보안 도구 모음 성격이 강함 | Debian 기반으로 재정리된 보안 배포판 |
| 실습 기준 | 옛 강의/책에서 등장 가능 | 지금 실습은 보통 Kali 기준 |
> [!note] 수업 환경
> 일반적으로 BackTrack은 Kali Linux의 전신이고 현재 표준은 Kali Linux다.
> 하지만 이 수업에서는 강사님이 배포한 BackTrack VM을 기준 환경으로 사용한다.
> 이유는 최신 Kali보다 구형 실습 도구와 취약한 프로토콜 실습이 강의자료대로 재현될 가능성이 높기 때문이다.
>
> 따라서 실무 기준으로는 Kali를 기억하되, 수업 실습 재현 기준으로는 BackTrack VM을 우선한다.
> [!important] 핵심
> BackTrack과 Kali는 완전히 다른 목적의 OS라기보다, **BackTrack이 Kali로 이어진 것**에 가깝다.
> 오래된 자료에서 BackTrack이 나오면 “아, 지금 기준으로는 Kali 쪽 이야기구나”라고 보면 된다.

> [!warning] 주의
> BackTrack은 최신 보안 실무 환경이라기보다 수업 재현용 레거시 실습 환경으로 본다.
> 실제 업무나 최신 도구 학습은 Kali Linux 기준으로 보는 편이 맞다.

> [!warning] 주의
> Kali는 해킹 마법 도구가 아니라, 보안 실습 도구가 많이 들어 있는 Linux 배포판이다.
> 기본 Linux 사용법을 모르면 Kali 자체보다 네트워크 설정, 권한, 패키지 관리에서 먼저 막힌다.
