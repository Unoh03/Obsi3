스니핑

	- 스위치(Switch): 원래는 안 보임 → 다른 공격 필요
- 대응 방법 : SSL/TLS와 같은 암호화 환경
스니핑 종류
❖ Active Sniffing 
- 패킷 흐름을 공격자가 의도적으로 조작
- 스위치(Switch) 환경에서 사용대표 기법: 
	- ARP Spoofing (지금까지 활용됨)
		- 공격자가 자신의 MAC 주소를 Gateway의 MAC 주소로 위조하여 ARP Cache를 오염 시키는 공격으로, 로컬 네트워크의 트래픽을 공격자가 중간에서 가로챌 수 있게 만 든다.
		- 동작 원리: 공격자가 자신의 MAC 주소를 Router(Gateway)의 MAC 주소인 것처럼 위조 → 위조된 ARP Reply를 Broadcast 또는 Unicast로 전송 → 로컬 네트워크 시스템들의 ARP Cache Table이 오염(Poisoning) 됨
		- 방어법: 통신 암호화 해라.
	- ~~MAC Flooding~~ (지금 못 씀)
		- 스위치의 MAC 주소 테이블(CAM Table)을 고의로 가득 채워 스위치를 허브(Hub)처 럼 동작하게 만드는 공격 기법
		- MAC Address 제한, Storm Control 등으로 역사적인 기법으로 남음.
		- 동작 원리: MAC 주소를 디도스마냥 왕창 보내면 테이블이 포화돼서 스위치가 그냥 브로드캐스트 함. 마치 Hub 처럼.
	- ~~ICMP Redirect~~ (지금 못 씀)
		- ICMP Redirect 메시지를 위조하여 피해자의 라우팅 경로를 공격자 쪽으로 변경하는 공격
		- 대부분의 OS는 ICMP Redirect 비활성화. 아예 기능이 안만들어져 있는 경우도 있음.
		- 동작 원리: 공격자가 피해자에게 “더 좋은 경로가 있다”는 가짜 ICMP Redirect 메시지 전송 → 피해자는 해당 메시지를 신뢰하고 라우팅 테이블을 변경 → 이후 트래픽이 공격자를 경유
❖ Passive Sniffing
- 패킷을 변조하지 않고 관찰만 수행 (와이어샤크로 가능.)
- 허브(Hub) 환경에서 가능공격 흔적이 거의 없음
  
이런 취약점을 진단하기 위한 도구들을 모은 것이 [칼리리눅스](https://www.kali.org/get-kali/#kali-virtual-machines).

데이터 암호화 (Data Encryption)
- Sniffing은 평문(Plain Text) 통신에서만 효과적
- 암호화를 적용하면 패킷을 가로채더라도 내용 해석 불가
- 대표 기술
	- SSL/TLS : HTTPS 기반 웹 통신 암호화
	- SSH: Telnet을 대체하는 안전한 원격 접속
	- VPN: 네트워크 구간 전체를 암호화하여 보호
- 암호화는 Sniffing 공격에 대한 가장 근본적인 대응 방법

❖ 스위치 환경 보안 구성
- Static ARP 설정
	- ARP Cache를 정적으로 고정
	- ARP Spoofing으로 인한 ARP Cache 변조 방지
	- 중요 서버 및 Gateway에 적용
- DHCP Snooping
	- 신뢰되지 않은 포트에서의 DHCP 응답 차단
	- 가짜 DHCP 서버 공격 방어
	- DHCP Snooping Database를 기반으로IP–MAC 매핑 관리
	- ARP Spoofing / MITM 공격의 기반 차단
- Dynamic ARP Inspection (DAI)
	- ARP 패킷의 IP–MAC 매핑을 검증
	- DHCP Snooping 테이블과 비교
	- 위조된 ARP 패킷 차단
	- ARP Spoofing에 대한 가장 강력한 스위치 방어 기법
- IP Source Guard
	- 포트별로 허용된 IP–MAC–Port 조합만 통과
	- IP Spoofing 방어DHCP Snooping과 연계 사용
