# 웹보안 총정리 미니 프로젝트 - 작업 메모

이 파일은 `웹보안 총정리 미니 프로젝트.md`를 정리하는 동안 Codex가 먼저 읽을 프로젝트 전용 작업 메모다.
이 파일 자체는 강사님께 보낼 PDF 내보내기 대상에 포함하지 않는다.

## Codex 진행 원칙

- Codex는 선택을 확정하지 않는다. 구조, 선택지, 장단점, 추천안을 제안한다.
- 파일 생성, 삭제, 수정, memory 기록은 사용자 명시 승인 후에만 실행한다.
- 사용자의 `좋다`, `대찬성` 같은 반응을 바로 실행 승인으로 해석하지 않는다.
- 실행 전에는 어떤 파일에 어떤 내용을 넣을지 먼저 전부 보여주고 확인받는다.
- 이상한 띄어쓰기, 오타, 용어 충돌, 링크 깨짐 가능성은 발견하면 지적한다.

## 프로젝트 노트 기준

- `웹보안 총정리 미니 프로젝트.md`는 사용자가 보는 노트이기도 하다.
- 평소 다른 Obsidian 노트처럼 가독성 있게 정리하고, 그 상태를 `PDF로 내보내기` 기능으로 강사님께 보낸다.
- 이미지와 callout은 PDF 내보내기에서 잘 보인다면 적극 활용할 수 있다.
- 다만 링크처럼 PDF에서 깨지거나 독자가 이해하기 어려울 수 있는 요소는 미리 처리 방식을 정한다.
- 템플릿은 아직 고정하지 않는다. 최소 2~3개 항목의 실제 프로젝트 데이터가 쌓인 뒤 반복 구조를 추출한다.

## 환경 구분

- 이번 대상은 강사님이 제공한 Win 2000의 Camel 쇼핑몰 서버다.
- 기존 Apache/PHP `care` 실습 환경과 Camel 환경을 섞어 쓰지 않는다.
- 기존 실습 노트는 Camel 절차서가 아니라 원리와 정리 구조를 참고하는 자료다.
- 경로, 파일명, 서버 종류, 계정, 포트, payload, 응답 화면은 Camel 환경에서 다시 확인한다.
- Camel에서 직접 확인하지 않은 결과를 성공처럼 쓰지 않는다.

## 링크 방식

기준 자료 링크는 GitHub 파일 링크만 사용한다.
GitHub permalink와 하단 링크 모음 방식은 사용하지 않는다.

### GitHub 파일 링크

GitHub의 현재 파일 경로를 가리키는 링크다.

예상 형태:

```text
https://github.com/<owner>/<repo>/blob/<branch>/<path>
```

장점:

- 강사님이 브라우저에서 바로 열 수 있다.
- Obsidian을 몰라도 링크 의미가 비교적 분명하다.
- 현재 최신 파일을 보여주고 싶을 때 자연스럽다.

주의점:

- repository가 private이면 강사님에게 접근 권한이 있어야 한다.
- branch의 파일이 나중에 바뀌면, PDF에 남은 링크도 바뀐 내용을 보여줄 수 있다.
- 파일명이나 경로를 바꾸면 링크가 깨질 수 있다.

## 현재 미결정 사항

- 6개 항목이 쌓인 뒤 공통 템플릿을 만들지, 항목별 구조를 유지할지.
