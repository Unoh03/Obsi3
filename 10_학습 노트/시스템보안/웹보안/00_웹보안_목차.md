## 현재 재시작 지점

1. Camel 종합 복습 프로젝트의 핵심 실습은 [[10_학습 노트/시스템보안/웹보안/장운호_웹보안실습|장운호_웹보안실습]]에 정리 완료했다.
2. 다음에 새 stable concept note를 만든다면 `SQL Injection 방어`를 1순위로 본다.
   - 기존 SQL Injection 공격·정보 추출 노트는 있다.
   - 방어 노트는 아직 없다.
   - 시작 source는 [[10_학습 노트/시스템보안/웹보안/SQL Injection 페이지별 분해 기록|SQL Injection 페이지별 분해 기록]]과 [[10_학습 노트/시스템보안/웹보안/SQL Injection Error와 UNION 기반 정보 추출과 Schema 파악|SQL Injection Error와 UNION 기반 정보 추출과 Schema 파악]]이다.
   - PDF 대응책만 옮기지 말고 Prepared Statement / Parameterized Query 중심으로 보강한다.
3. 그다음 후보는 `세션과 쿠키`다.
   - source 범위는 [[10_학습 노트/시스템보안/웹보안/5-20 웹보안 PDF 구조 지도|5-20 웹보안 PDF 구조 지도]]의 p.49-61이다.
   - [[10_학습 노트/시스템보안/웹보안/Web Session Hijacking|Web Session Hijacking]], [[10_학습 노트/시스템보안/웹보안/XSS|XSS]], [[10_학습 노트/시스템보안/웹보안/CSRF|CSRF]]를 이해하는 기반 노트로 만든다.
4. 나머지 미작성 기반 주제는 후속 후보로 유지한다.
   - 웹 문서, DOM, 웹 언어
   - URL Encoding, Unicode, Base64. 단, [[10_학습 노트/시스템보안/웹보안/HTML 인코딩|HTML 인코딩]]은 이미 있으나 draft 상태다.
   - 기초 웹 인증 방식 - Basic, Anonymous, Form
   - OWASP Top 10 2025. 작성 시점에 OWASP 공식 자료로 확인한다.

## 노트 품질 기준

- PDF page는 출처로만 두고, 본문은 PDF 없이 읽히는 독립 개념 노트처럼 작성한다.
- 중요한 노트에는 `수업 표현을 정확한 개념으로 바꾸기`, `오해하기 쉬운 지점`, `확인 질문`을 둔다.
- raw 메모의 질문은 삭제하지 않고 `질문 -> 해석 -> 답` 흐름으로 흡수한다.
- 외부 정보는 공식 문서 우선으로 확인하고, 바뀔 수 있는 서비스/가격/보안 기능은 작성 시점 기준을 남긴다.
