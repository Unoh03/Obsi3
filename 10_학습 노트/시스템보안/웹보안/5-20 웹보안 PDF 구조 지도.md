# 5-20 웹보안 PDF 구조 지도

source: [[40_자료/강의 자료/5-20_웹보안.pdf|5-20 웹보안]], p.1-136
raw memo: [[10_학습 노트/시스템보안/서론(이라쓰고 빠르게 휘갈겨 쓴거)|서론 raw 메모]]

## 목적

이 노트는 `5-20_웹보안.pdf`를 바로 요약하기 위한 노트가 아니라, 앞으로 세부 정리 노트를 만들 때 사용할 **page range와 주제 경계 지도**다.

작성 기준:

- 1순위 근거: PDF에서 추출한 page heading과 단원 번호
- 2순위 근거: raw 수업 메모의 질문과 강조
- 3순위 근거: 기존 웹보안 목차와 이미 작성한 노트의 cross-check

기존 노트를 기준으로 PDF를 끼워 맞추지 않는다. 충돌이 있으면 이 구조 지도에 `기존 목차와 충돌 가능`으로 표시하고, 노트 작성 전에 다시 판단한다.

---

## 구조 지도 정책

- 이 지도는 최종 노트 목록이 아니라 PDF의 중간 단위 주제 경계 지도다.
- PDF의 단원 순서는 유지한다.
- 하나의 PDF 단원이 실제 노트로는 너무 크면, 행을 무조건 쪼개지 않고 `노트화 시 분리 후보`를 분리/병합 판단에 표시한다.
- 단, PDF 안에서 별도 heading이 있고 원인·영향·대응이 다른 취약점은 중간 지도에서도 하위 분리로 명시한다.
- 최신성 또는 공식 기준이 중요한 내용은 PDF를 1차 강의 자료로 보되, 공식 문서로 정정/보강한다.
- 공격 예시는 실습·방어 이해 목적의 최소 맥락으로만 남긴다.

## 기존 목차와 다른 점

- `HTTP`는 PDF 기준으로 p.4-23 한 단원이지만, 현재 노트는 `HTTP 구조와 메시지`와 `HTTP Method와 Header`로 분리되어 있다. 충돌이라기보다 학습 편의상 좋은 분리다.
- 기존 목차의 `HTML DOM과 클라이언트 스크립트`는 PDF p.24-34 전체를 담기에는 제목이 좁을 수 있다. 이 범위에는 ASP, ASP.NET, PHP, JSP 같은 server-side script도 들어간다.
- 기존 목차의 `Directory Indexing`은 PDF 표기상 `Directory Listing 취약점`이다. 같은 취약점을 가리키는 표현으로 보되, 노트 제목은 기존 목차 표현을 유지해도 된다.
- `세션과 쿠키`는 기존 목차와 잘 맞지만, raw 메모와 PDF 모두 비중이 크다. 노트 작성 시 단독 중요 노트로 다루는 것이 맞다.
- `웹 인증 구조`는 기존 목차 제목으로는 유지 가능하지만, 실제 노트 제목은 PDF 범위에 맞춰 `기초 웹 인증 방식 - Basic, Anonymous, Form`을 우선 검토한다.
- `ID/PW Brute Forcing`은 공격명으로는 맞지만, 실제 노트 제목은 방어 관점을 살려 `로그인 Brute Force와 계정 보호`가 더 적절하다.

## 작성 시 주의

- 이 구조 지도는 노트 본문이 아니다. 각 세부 노트 작성 시 해당 page range를 다시 읽는다.
- PDF page는 출처로 남기고, 본문 제목은 개념 중심으로 쓴다.
- raw 메모 질문은 삭제하지 않고 `질문 -> 해석 -> 답`으로 흡수한다.
- 공격 예시는 학습/방어 맥락으로 제한한다.
- 최신성이 필요한 주제는 작성 시점에 공식 문서를 다시 확인한다.
